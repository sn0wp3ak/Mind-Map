  • Django
    • 安装+创建+启动
      • 创建: django-admin startproject 项目名
      • 启动: python manage.py runserver [ip:port]
        • Django默认启动IP: 127.0.0.1 端口号: 8000
      • 安装: pip install django
    • 项目目录结构
      • 项目根目录
        • 同名子目录
          • __init__.py
          • settings.py
          • urls.py
          • wsgi.py
        • manage.py
    • 子应用
      • 创建
        • python manage.py startapp 子应用名
      • 注册
        • 在settings.py的INSTALLED_APPS列表中添加
        • 注册格式: '子应用名.apps.apps文件中的Config类'
      • 子应用目录结构
        • __init__.py
        • admin.py
        • apps.py
        • migrations/
          • __init__.py
        • models.py
        • test.py
        • views.py
    • 视图
      • 创建
        • 在子应用的views.py中编写视图代码
    • 路由
      • 主路由文件
        • 项目同名子目录中的urls.py文件(路由解析的入口)
      • 子路由文件
        • 在子应用目录下新建urls.py文件
      • 路由路径: 主路由找子路由, 子路由找视图
        • include方法用来把子路由包含到主路由 include('子应用.urls')
        • Django在检查路由时, 即使不加/也会自动定向到带/的路由
      • 路由命名(给路由起名)
        • 在使用include函数时, 使用namespace定义命名空间
          • 一般是给子应用定义命名空间
        • 在子路由urls文件的列表使用name参数定义具体路由名字
        • 作用: 方便查找, 解决路由屏蔽问题
      • 路由解析
        • 顺序:由上至下(可能会导致路由屏蔽)
        • 路由屏蔽的解决方案
          • 刻意地调整路由定义的顺序
          • 使用命名来规划路由
        • 反向解析
          • 根据命名返回具体的路由路径
          • 格式: url = reverse('namespace:name')
